❌
Write a wall of text - agents read instructions, not essays
❌
Auto-generate it with AI - hand-curated beats AI-generated
🏠 Monorepo Hierarchy

In a monorepo, agents use the AGENTS.md closest to where they're working.

my-monorepo/
├ AGENTS.md          # root: global rules
├ packages/
│  ├ api/
│  │  └ AGENTS.md   # api-specific
│  ├ web/
│  │  └ AGENTS.md   # web-specific
│  └ shared/
└ README.md
  • ✦Root AGENTS.md = global rules for the whole repo
  • ✦Subdirectory AGENTS.md = overrides or adds rules for that package
  • ✦Subdirectory rules win when they conflict with root rules

📋 The Meta-Tip

Found yourself correcting your agent again on the same thing?

"Please update AGENTS.md with this rule so you remember it next time."

This is the fastest way to evolve your AGENTS.md over time. Let every correction become a permanent rule.

⚠️ Common Mistakes
  • ✗Bloating it - adding every possible rule. Long = ignored.
  • ✗Generic tips - "write clean code". Agents already know this.
  • ✗Stale commands - build command changed, AGENTS.md didn't.
  • ✗Missing constraints - not saying what NOT to touch. Most critical omission.
  • ✗One monolithic file - in monorepos, use the hierarchy.
