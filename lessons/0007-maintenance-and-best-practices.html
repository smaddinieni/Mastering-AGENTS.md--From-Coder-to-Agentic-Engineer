Maintenance and Best Practices

Keep your AGENTS.md file short, accurate, and powerful as your codebase grows.

~8 min read 3 quiz questions Advanced
Your progress 90% Complete
Rule 1

Keep the File Short and Focused

An AI coding agent reads your entire AGENTS.md file at the start of every session. If the file is too long, it wastes valuable context space. This makes the agent slow, increases API costs, and confuses its logic.

To prevent this, you must only list rules that the agent cannot find by looking at your code. Do not write long guides. Write simple, direct facts.

Incorrect Approach

Long and Redundant Rules

  • Explaining how the React framework works.
  • Adding long code snippets of normal helpers.
  • Adding rules that are checked by eslint files.
Correct Approach

Short and Direct Guidelines

  • "Use standard CSS instead of Tailwind libraries."
  • "Never edit the /vendor folder."
  • "Run pnpm build to verify type safety."
Rule 2

Prevent Stale Rules and Commands

A stale rule is an instruction that does not match your codebase anymore. For example, if you change your test tool from Jest to Vitest, but you do not update the test command in your AGENTS.md file, the agent will try to run the old command. This leads to failures.

You must clean up your rules. Whenever you update your build tools, rename folders, or change your design patterns, you must update the AGENTS.md file immediately. Remove rules that are no longer valid.

!
Warning about stale rules

If the agent reads a stale command, it will waste attempts trying to run it. This wastes your time and breaks the automated build process.

The Meta-Tip

Update Rules When Correcting the Agent

When pair programming with an agent, you will sometimes need to correct its behavior. For example, you might say, "Please do not use double quotes for imports, we use single quotes."

The agent will fix the issue in that chat session. But when you start a new session, it will forget this instruction and make the same mistake again.

The Meta-Tip: The moment you have to correct the agent for the same mistake twice, do not just type the fix in chat. Open your AGENTS.md file and add a simple rule.

AGENTS.md Markdown 
## Code Style Rules

- Always use single quotes for JS and TS imports.
- Keep all helper functions inside the utils folder.

By doing this, you update the agent's memory for all future sessions. You teach it once, and it remembers forever.

Pitfalls

Common Pitfalls to Avoid

Watch out for these common errors when writing and maintaining your rules:

1. Redundant Rules: Do not write rules that are already defined in config files. The agent can check files like package.json, tsconfig.json, and eslint configs. Let the files speak for themselves.

2. Soft Guidelines: Rules like "write clean code" or "be efficient" do not help. AI agents need concrete limits and details. Write "Keep functions under 50 lines" instead.

3. Too Many Rules: If your file gets longer than 100 lines, it is probably too long. Keep the list of rules small so the agent can prioritize them correctly.
